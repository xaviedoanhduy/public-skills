# Odoo Module Structure Reference

## Standard OCA-Compliant Directory Tree

```text
<module_name>/
├── __init__.py                         # REQUIRED — imports models, controllers, wizards
├── __manifest__.py                     # REQUIRED — module metadata
│
├── models/                             # REQUIRED if module has business logic
│   ├── __init__.py                     # imports all model files
│   └── <model_name>.py                 # one file per model class
│
├── views/                              # REQUIRED if module has UI
│   ├── <model_name>_views.xml          # form + list views per model
│   └── menus.xml                       # ir.ui.menu entries
│
├── security/                           # REQUIRED for any model
│   ├── ir.model.access.csv             # ACL: model-level read/write/create/unlink
│   └── security.xml                    # res.groups definitions, record rules (noupdate="1")
│
├── data/                               # optional — master/config data loaded on install
│   └── <name>.xml
│
├── demo/                               # optional — demo data (omit with --no-demo)
│   └── demo_data.xml
│
├── controllers/                        # optional — HTTP routes
│   ├── __init__.py
│   └── main.py
│
├── wizard/                             # optional — transient models
│   ├── __init__.py
│   └── <wizard_name>.py
│
├── report/                             # optional — QWeb PDF reports
│   ├── <report_name>_report.xml        # ir.actions.report record + QWeb template
│   └── <report_name>_report.py        # Python model if report needs custom data
│
├── tests/                              # REQUIRED (OCA mandatory)
│   ├── __init__.py
│   └── test_<module_name>.py           # TransactionCase-based tests
│
├── i18n/                               # optional — translations
│   └── <module_name>.pot               # POT template (generated by Odoo)
│
├── static/                             # optional — frontend assets
│   ├── description/
│   │   ├── icon.png                    # 64x64 PNG, shown in Apps menu
│   │   └── index.html                  # module description page
│   └── src/
│       ├── components/                 # OWL components
│       │   ├── <component>.js
│       │   ├── <component>.xml
│       │   └── <component>.scss
│       └── js/
│           └── <widget>.js
│
└── readme/                             # OCA contribution standard
    ├── DESCRIPTION.rst                 # module description (auto-generates README.rst)
    ├── INSTALL.rst                     # installation notes (optional)
    ├── CONFIGURE.rst                   # configuration steps (optional)
    ├── USAGE.rst                       # usage instructions
    └── CONTRIBUTORS.rst                # contributor list
```

> For QWeb report templates, portal templates, and OWL component templates,
> see `references/qweb-templates.md`.

## File Generation Order

Generate files in this order to avoid import errors:

1. `__manifest__.py` — needed first to declare dependencies
2. `__init__.py` — top-level, imports everything
3. `models/__init__.py`
4. `models/<model_name>.py`
5. `security/security.xml`
6. `security/ir.model.access.csv`
7. `views/<model_name>_views.xml`
8. `views/menus.xml`
9. `tests/__init__.py`
10. `tests/test_<module_name>.py`
11. Optional dirs: `data/`, `demo/`, `controllers/`, `wizard/`, `i18n/`, `static/`, `readme/`

## Naming Conventions

| Item | Convention | Example |
|---|---|---|
| Module directory | `snake_case` | `sale_custom_approval` |
| Model class | `PascalCase` | `SaleCustomApproval` |
| Model `_name` | `dot.notation` | `sale.custom.approval` |
| XML IDs | `module.model_description` | `sale_custom.view_custom_form` |
| Security group | `module.group_module_role` | `sale_custom.group_approval_manager` |
| Menu XML ID | `module.menu_model_list` | `sale_custom.menu_approval_list` |
| View XML ID | `module.view_model_form` | `sale_custom.view_approval_form` |
| Action XML ID | `module.action_model` | `sale_custom.action_approval` |

## Module Name Validation

- Must match: `^[a-z][a-z0-9_]+$`
- No uppercase, no hyphens, no spaces
- Must not start with a digit
- Recommended prefix = Odoo functional area: `sale_`, `account_`, `hr_`, `stock_`, etc.
